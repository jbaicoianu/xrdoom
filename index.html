Spispopd.lol is an online viewer for Doom and Heretic maps. It is able to load .wad files, extract their resources, and render them using JanusWeb, which gives us a way to explore these worlds in XR.



December 20, 2019
Updated to work with latest Three.js
   Looks like things broke recently when we upgraded to a newer version of Three.js, which renamed some functions. Oops! It's fixed now, and so is the HUD and mini-map. Enjoy!

February 26, 2018
Now working in Firefox 65
   As mentioned in the previous post, Firefox has fixed their ES6 Module bug, and with the release of Firefox 65, WebVR Doom now runs in every major browser! Woohoo!

   Now we're just waiting on Edge...their slow approach to adopting web standards means that we've used some functions they haven't yet implemented. Apparently Edge 18 fixes at least one of these issues - so now I just have to figure out how to actually upgrade. That, or wait them out until they release their newly-announced Chromium-based version...

-- bai

December 6, 2018
Firefox Nightly support!
   Hooray! I'm excited to announce that we were able to identify the bug which was preventing WadJS from running in Firefox, and once we had a simplified test case, the team over at Mozilla was able to provide a fix in a matter of hours. That fix has already made it into the latest Nightly builds, meaning that you can now use WadJS to view Doom levels in Firefox Nightly, and that this fix should make its way into a release version very shortly. Thanks, Mozilla!

September 25, 2018
Alpha update!!
   Hot on the heels of JanusWeb 1.3, we're now making our HTML Doom level viewer public alpha available for everyone to test! We've been looking forwrd to this release for a while now, we've fixed some of the bugs but there are still plenty left to find - both in our code, and in the browsers (I'm looking at you, Mozilla! And, well...you too, Edge, but we'll deal with you later.)

   If you're interested in how this all works, we're working on a full write-up, but suffice to say we're using some cutting-edge browser technology here - ES6 Modules, HTML5 WebComponents, WebGL/WebVR - the works! The code for loading WAD files in JS is available on Github, we'll be publishing the client code along with the technical write-up soon - keep an eye out for that in the next couple weeks.

-- bai
